# Sortie rapide + revenus — Le Bar Zine

## Objectif

Sortir vite une webapp utile, puis créer des premiers revenus sans perdre l'esprit zine, local, underground, alternatif et éditorialement situé.

Le produit n'est pas seulement un guide de bars. C'est un radar local, avec une ligne plutôt gauche culturelle / indépendante :

- où aller selon ton mood ;
- quoi suivre dans la scène locale ;
- quels lieux et collectifs méritent d'être relayés ;
- comment soutenir une veille indépendante.

## MVP public

À publier en premier :

1. Guide bars + moods.
2. Agenda / appels à events.
3. Bouton "Proposer un event".
4. Page "Soutenir" avec économie transparente.
5. Badges de fiabilité éditoriale.

Ce qui peut attendre :

- compte utilisateur ;
- backend ;
- carte interactive ;
- scraping automatique ;
- paiement intégré maison.

## Revenus rapides, sans vendre la ligne

### 1. Soutien lecteur

Brancher un lien Ko-fi, Buy Me a Coffee, Patreon, GitHub Sponsors, Twint ou équivalent.

Message clair : le soutien finance la veille locale, les sorties terrain, l'hébergement et la mise à jour du zine.

### 2. Édition soutenue

Proposer une présence clairement marquée :

- "édition soutenue par..." ;
- logo discret ;
- lien vers un lieu, festival, disquaire, asso ou média local ;
- jamais de classement acheté.

Prix de départ possible :

- 50 CHF pour une mini mise en avant ponctuelle ;
- 100-150 CHF pour une édition locale soutenue ;
- 250 CHF+ pour un pack lieu / événement / mini page.

### 3. Services éditoriaux locaux

Offrir aux lieux et collectifs :

- mini fiche propre ;
- mini-site événement statique ;
- page agenda simple ;
- kit de liens pour bio Instagram ;
- reprise visuelle style zine.

Ce sera probablement plus rentable au début que la pub classique.

## Ligne éditoriale

À protéger :

- culture locale ;
- lieux alternatifs ;
- scènes underground ;
- événements politiques, associatifs, militants ou indépendants ;
- bars et lieux hybrides qui accueillent vraiment quelque chose.
- transparence sur l'argent ;
- priorité aux lieux sans grosse force de communication.

À éviter :

- devenir Google Maps ;
- vendre les recommandations ;
- vendre une ligne éditoriale ;
- remplir l'agenda avec n'importe quelle soirée commerciale ;
- présenter une info non vérifiée comme un fait.

## Prochaine action avant publication

Remplacer les liens temporaires :

- `SUPPORT_URL` dans `assets/app.js` ;
- `CONTRIBUTION_URL` dans `assets/app.js` si tu préfères un formulaire plutôt que GitHub Issues.

Ensuite : publier GitHub Pages et partager une première version "beta zine".
